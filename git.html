Introduction

Git adalah salah satu sistem pengontrol versi (Version Control System) pada proyek perangkat lunak yang diciptakan oleh Linus Torvalds. Sebelum membahas git, kita harus mengenal terlebih dahulu tentang VCS (Version Control System).

Apa itu Version Control System (VCS)?

Menurut Atlassian, version control system (VCS) adalah sebuah kumpulan perangkat lunak yang sudah terintegrasi dan digunakan untuk membantu software engineer mengelola perubahan dalam source code dari waktu ke waktu.

Version Control System atau VCS, juga dikenal sebagai sistem pengendalian versi atau sistem manajemen versi, adalah perangkat lunak yang digunakan untuk melacak perubahan dalam kode sumber dan dokumen lainnya selama pengembangan perangkat lunak. Tujuan utama dari VCS adalah untuk memungkinkan kolaborasi antara anggota tim pengembangan, melacak perubahan yang dilakukan oleh setiap anggota tim, dan memungkinkan pemulihan kode sumber ke versi sebelumnya jika diperlukan. Nah sekarang kita akan membahas salah satu VSC yaitu git.

Apa itu Git?

GIT merupakan tools yang biasa digunakan para programmer dan developer sebagai control system dalam menjalankan proyek pengembangan software. GIT sendiri merupakan singkatan dari Group Inclusive Tour, sebuah version control system yang dikembangkan oleh Linus Torvalds pada 2005.

GIT dikategorikan sebagai DVCS atau Distributed Version Control System. Artinya, kode tidak hanya memiliki satu tempat penyimpanan history lengkah sebuah software. Dengan GIT, setiap salinan kode kerja developer juga berperan sebagai repositori yang bisa berisi riwayat lengkap dari semua perubahan yang sudah dilakukan. Selain itu, GIT juga dirancang untuk kinerja, fleksibilitas, dan keamanan yang maksimal. GIT memiliki beberapa tool, berikut adalah tools dari GIT :

Command atau Perintah pada Git

        - git init     : git init berfungsi untuk membuat repository kosong yang baru atau mengubah folder yang telah ada menjadi sebuah git repository.
        - git status   : git status berfungsi untuk mengecek status git.
        - git add      : git add berfungsi untuk menambahkan perubahan yang telah Anda buat dalam direktori kerja Anda ke dalam area staging (staging area) 
        sehingga perubahan tersebut siap untuk di-commit (dikommitkan) ke dalam repositori Git Anda. dalam git add terdapat git add . dan git add nama_file :
        git add .      : berfungsi untuk menambahkan seluruh perubahan yang belum di stage.
        git add name nama_file : berfungsi untuk menambahkan suatu file ke dalam stage.
        - git log      : git log berfungsi untuk menampilkan riwayat commit pada repository.
        - git commit   : git commit berfungsi untuk menyimpan perubahan yang udah ditambahkan kedalam repository
        - git checkout : git checkout berfungsi untuk beralih ke branch atau cabang commit, dengan menggunakan code : git checkout (folder branch/cabang).
        - git branch   : git branch berfungsi untuk membuat branch baru dengan menggunakan code git branch (nama branch yang diingin).
        - git push     : git push berfungsi untuk mengirim perubahan atau commit di repository lokal kedalam repository git seperti GitHub.
        - git pull     : git pull berfungsi untuk membuat perubahan dengan mengambil perubahan terbaru dari server jarak jauh dan digabung kedalam branch lokal.
